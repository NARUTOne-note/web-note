# web安全

## 防御性编程

页面在呈现给用户之前需要经过静态资源加载、后端接口请求和渲染这三个过程，我们要做的就是在各个过程中防御可能出现的异常情况，保持流畅的用户体验，同时还要应对来自外部的攻击

1、**防网络**

- 网络较差数据返回慢，页面不渲染、空白页：loading、骨架屏等

2、**防接口**

- 超时：超时限制，提升信息
- 错误返回：针对不同 code 码，提升信息、跳出登录等
- 业务错误：提示信息
- 请求取消

3、防渲染

- 异常渲染：null、undefined、属性报错等
- 降级渲染：代码报错，降级渲染

4、防重

- 按钮重复点击：loading
- 滚动、输入：防抖

5、防攻击

- xss
- csrf

6、防依赖库漏洞

- CVE 安全平台报告
- npm audit fix 漏洞检查修复
